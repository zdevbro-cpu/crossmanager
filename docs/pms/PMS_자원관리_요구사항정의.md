아래는 지금까지 합의된 개념( **인력 원장/민감정보는 "시스템관리(보안영역)"**, PMS 자원모듈은 **배정에 필요한 최소정보/판정값만 소비** )을 전제로 한, **시스템관리 > 인력관리(현장투입·엔지니어링 인력 중심)** 기능요구사항 정의서 초안입니다.  
※ 급여/근태/인사발령 등 "전통 HR" 영역은 의도적으로 제외합니다(중복·민감도·범위 증가 방지).

**1) 모듈 목적**

- 현장 투입 전 **자격/교육/건강·안전 요건 충족 여부**를 신뢰성 있게 관리하고,
- PMS 자원모듈에서 **배정 가능/불가 판정**과 \*\*유효기간(만료 D-day)\*\*를 즉시 활용할 수 있게 한다.
- 개인정보는 최소 수집·최소 노출(권한 기반/마스킹)로 운영한다.

**2) 범위 정의**

**2.1 포함 범위(현장/엔지니어링 중심)**

- 인력 마스터(현장투입 인력, 엔지니어링 인력, 협력사 인력 포함)
- 직무/역량/등급(기술분야, 수행 가능 작업군)
- 자격증/면허/수료증 관리(유효기간, 증빙파일, 검증 상태)
- 안전/법정 교육 이력(필수교육, 특수작업 교육 포함) 및 만료 알림
- 투입 가능 판정(룰 기반) + 예외 승인(승인 이력 포함)
- 프로젝트/현장 배정 시 "필수 요건 충족" 자동 검증을 위한 **판정값 제공(API/뷰)**

**2.2 제외 범위(명시적 제외)**

- 급여/정산, 4대보험, 인사발령/평가, 근태(출퇴근), 사무직 인력관리 전반
- 채용/온보딩 HR 프로세스 전체(필요 시 최소 필드만 연동)

**3) 사용자/권한(Role 기반)**

- **TENANT_ADMIN**: 전체 설정/권한/코드 관리
- **HR_SEC_ADMIN(인력보안관리자)**: 민감정보/증빙파일 접근, 등록/수정 승인
- **SAFETY_ADMIN**: 교육·자격 기준/템플릿, 만료 알림 정책
- **PROJECT_MANAGER / SITE_MANAGER**: 프로젝트 배정 시 판정값 조회, 예외요청 생성
- **PMS_VIEWER(소비자)**: PMS에서 필요한 최소 필드/판정값만 조회(원장 수정 불가)

핵심: **민감정보/증빙 원본 접근은 HR_SEC_ADMIN만**, 현장관리자는 "충족/미충족" 중심으로.

**4) 기능 요구사항 (FR)**

**PIM-FR-100 인력 마스터 관리**

- **FR-101 인력 등록/조회/비활성화**
  - 대상: 현장투입/엔지니어링/협력사 인력
  - 상태: active / inactive / blocked(투입 금지)
- **FR-102 최소 프로필(배정용) vs 민감 프로필(보안용) 분리**
  - 배정용: 이름(또는 이니셜), 소속/협력사, 직종/등급, 기술분야, 작업가능군, 재직상태, 프로젝트 투입 가능여부(판정값)
  - 보안용: 상세 연락처, 주민/여권 등 고유식별(필요 시), 주소, 자격증 원본 파일, 계약/개인서류(필요 시)
- **FR-103 중복 방지**
  - 휴대폰/이메일/사번(있다면) 기준 중복 등록 탐지 및 병합 요청 프로세스

**PIM-FR-200 직무/역량(스킬) 및 작업가능군 관리**

- **FR-201 직무/기술분야/등급 코드 관리**
  - 예: 용접(등급), 전기, 배관, 신호수, 안전감시자, 품질, 공무, 시운전 등
- **FR-202 작업가능군 매핑**
  - 예: 고소작업 가능, 화기작업 가능, 전기작업 가능, 중량물 인양 신호 가능 등
- **FR-203 경력/프로젝트 수행이력(선택)**
  - HR 수준 상세 이력 대신, "현장 투입 판단에 필요한 요약"만(예: 동일 공종 경험 N회)

**PIM-FR-300 자격증/면허/수료증 관리**

- **FR-301 자격 유형(템플릿) 관리**
  - 필수 여부, 유효기간 규칙, 만료 알림(30/60/90일), 요구 문서(스캔/사진)
- **FR-302 개인별 자격 등록**
  - 자격번호, 발급일/만료일, 발급기관, 증빙파일 업로드
- **FR-303 검증(Verification) 상태**
  - pending / verified / rejected
  - 검증자, 검증일, 반려사유 기록
- **FR-304 만료/유효 자동 판정**
  - "현재 유효", "만료", "D-30" 등 상태를 자동 산출하여 PMS에 제공

**PIM-FR-400 교육 관리(현장/안전 필수 중심)**

- **FR-401 교육 유형(템플릿) 관리**
  - 법정/정기/특별/작업특화 교육 구분, 유효기간, 필수 대상 직무
- **FR-402 교육 이력 등록**
  - 교육일시, 교육기관/강사, 교육내용 요약, 참석확인(서명/증빙파일)
- **FR-403 교육 유효 판정**
  - 유효/만료, 만료 예정 알림
- **FR-404 SMS 연계(선택)**
  - SMS에서 생성되는 교육 세션/참석기록이 있다면 인력관리 원장으로 동기화(중복 입력 최소화)

**PIM-FR-500 투입 가능 판정(Eligibility Engine)**

- **FR-501 작업/공종별 투입 요건 정의**
  - 예: "고소작업" = 고소작업 교육 유효 + 특정 자격 보유
  - 예: "화기작업" = 화기작업 교육 유효 + 안전서약 확인
- **FR-502 인력별 투입 가능 여부 자동 계산**
  - 결과: eligible true/false, 미충족 사유 목록(부족 자격/만료/미검증)
- **FR-503 예외 승인 프로세스**
  - 현장관리자 예외요청 → HR_SEC_ADMIN/SAFETY_ADMIN 승인/반려
  - 승인 이력, 유효기간(예외는 기간 제한 권장)

**PIM-FR-600 PMS 자원모듈 소비를 위한 제공 기능(핵심)**

- **FR-601 배정용 조회 API/뷰 제공**
  - 인력 검색(직무/기술/등급/협력사/가용상태)
  - "배정 가능 판정값", "만료 D-day", "부족 요건" 제공
- **FR-602 최소정보 노출 정책**
  - PMS 자원모듈에서는 민감정보 필드 접근 불가
  - 연락처는 "프로젝트/현장 권한 + 추가 권한"이 있을 때만 마스킹 해제
- **FR-603 배정 결과 피드백(선택)**
  - PMS에서 실제 배정된 작업/프로젝트 정보를 인력 원장에 "최근 투입" 수준으로 기록(가용/경험치 보조)

**PIM-FR-700 감사/보안/개인정보(필수)**

- **FR-701 접근 제어**
  - 민감 프로필/증빙파일은 HR_SEC_ADMIN만
- **FR-702 감사 로그(Audit Trail)**
  - 누가/언제/무엇을 조회·변경·다운로드 했는지 기록
- **FR-703 증빙 문서 불변성(권장)**
  - 자격/교육 증빙은 "검증 완료(verified)" 이후 원본 수정 금지(정정은 개정본 추가)
- **FR-704 보관/삭제 정책**
  - 퇴사/비활성화 후 보관기간, 마스킹/삭제 기준(사내 규정에 맞춤)

**5) 메뉴/화면 구성(시스템관리)**

- 시스템관리
  - 인력관리
    - 인력 목록/검색(필터: 직무/기술/등급/협력사/유효상태)
    - 인력 등록/상세(탭)
      - 기본(배정용) 프로필
      - 자격증(등록/검증/만료)
      - 교육(이력/유효)
      - 투입 가능 판정(요건 충족표/부족 사유)
      - 첨부/증빙(보안권한자만)
      - 변경 이력/로그(보안권한자만)
    - 기준/템플릿
      - 자격 유형 관리
      - 교육 유형 관리
      - 작업/공종별 투입 요건(룰) 관리
    - 검증/승인함
      - 자격 검증 대기
      - 예외 승인 요청
